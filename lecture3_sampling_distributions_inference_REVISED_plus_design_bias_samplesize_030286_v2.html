Sampling distribution: Definition

  • sampling distribution: probability distribution of a statistic computed from all possible samples of a fixed size from a population

Key distinction(s):

  • population parameter: fixed, unknown constant [μ, p, λ]
  • sample statistic: random variable [\(\bar{X}\), \(\hat{p}\), etc.]
  • sampling distribution: distribution of the statistic across repeated samples

Sampling in biomedical research

Review: we rarely observe an entire population [i.e.: all patients, all tumors, all genomes] instead a sample is taken and generalized for statistical analysis

  • sampling, bias, and sample size determine whether your study’s conclusions are:

    • valid [not systematically wrong]
    • precise [narrow uncertainty]
    • reproducible [another cohort gives compatible results]
    • actionable [effects are detectable at clinically meaningful scales]

Sampling and Statistical Analysis

A core objective of statistical analysis is to determine:

Does this sample allow the researcher to learn something true about a population?

Three components determine the answer:

  1. sampling strategy: defines who your data stand for
  2. bias: determines whether your estimate is systematically distorted.
  3. sample size: determines how much random uncertainty remains

Sampling strategies

Sampling strategy:

  • pre-specified procedure for selecting observations from a defined target population
  • determines who can be selected, how selection occurs, and each unit’s inclusion probability

Importance in practice:

  • determines whether estimates [i.e.: prevalence, mean biomarker level, incidence rate] generalize to the target population
  • determines whether standard errors / confidence intervals are trustworthy
  • determines the feasibility and cost of data collection [i.e.: multi-site, registries, EHRs, biobanks]

Examples:

  • selecting patients from a hospital registry to estimate readmission risk
  • selecting biospecimens from a biobank to characterize a biomarker distribution
  • selecting sequencing samples from a case series to study mutation spectra

Sampling Strategies

  • simple Random Sample [SRS]: every set of n units has equal chance of selection

    • strength: clean inference
    • weakness: requires full sampling frame

  • stratified sampling: divide into strata [age, sex, ancestry, site] then sample within strata

    • strength: ensures subgroup representation and increases precision
    • note: disproportionate sampling may require weights

  • cluster/multistage sampling: sample clusters [hospitals/clinics] then sample individuals within clusters

  • systematic sampling: random start, then every \(k^{th}\) unit from an ordered list

Sample size considerations

sample size considerations:

  • structured decisions used to determine how many observational units are required within a sample
  • how the observations should be distributed across groups or strata so that statistical estimates are sufficiently precise and comparisons have adequate power

structured process used to determine if there is enough data to answer the scientific question reliably, and considers:

  • how many observational units [patients, specimens, encounters, variants] must be included
  • how those units should be allocated across groups or strata

sample size is determined by several factors such as:

  • parameter you want to estimate [mean, proportion, rate, hazard]
  • variability of the outcome
  • smallest clinically meaningful effect
  • sampling design [SRS, stratified, clustered]

Sample size considerations

Small sample size characteristics:

  • estimates are more variable
  • standard errors are larger
  • confidence intervals are wider
  • statistical tests have lower power
  • sampling distributions may not approximate normality

Implications:

  • results could be less stable across repeated samples
  • effect estimates may fluctuate substantially
  • type II error [missing a real effect] is more likely

Commonly found in:

  • rare disease studies
  • early-phase clinical trials
  • pilot studies

Sample size considerations

large sample sizes:

  • estimates are more precise
  • smaller standard errors
  • narrower confidence intervals
  • higher statistical power
  • sampling distributions approach normality

implications:

  • greater stability across repeated sampling
  • smaller effects become detectable
  • more reliable inference

Why Sample Size Matters

Data is collected to answer two types of questions:

  1. Estimation questions
    • What is the value of a biological parameter?
  2. Comparison questions
    • Do groups differ in a meaningful way?
  • sample size planning helps ensure that the data collected contain enough information to answer the research question

Sample Size in Practice

  1. precision: focuses on estimation accuracy
  • Goal: choose sample size so that a confidence interval is narrow enough for the research question
  • Core question: How uncertain is our estimate?
  • larger n → smaller standard error → narrower confidence interval → greater precision
  • example: estimating a treatment response rate within \(±3%\)

Example:

Estimate the response rate within ±3%

  • sample size is chosen so the confidence interval width approximates the true estimate

Interpretation:

  • estimate will vary across samples, but with sufficient \(n\) the uncertainty becomes small enough to answer the research question

Sample Size in Practice

2)Power: detecting differences between groups

  • goal is to compare groups or test a hypothesis
  • Core question: If the true effect exists, what is the probability we detect it?
  • choose n so that you can detect a clinically meaningful effect with high probability
  • answers: If the true effect exists, what is the probability we detect it?

Depends on factors such as:

  • effect size: smaller effects require larger n

  • variability: more variability requires larger n

  • outcome type: binary, continuous, count, survival

Sample Size in Practice

  1. design adjustments: : adjustments that ensure the study still contains enough effective information
  • design adjustments correct for:
  • independent observations
  • participants are naturally grouped [i.e.: patients grouped by hospital] > observations within a group/cluster tend to be correlated >> correlated observations contribute less independent information and reduces the effective sample size
  • solution: increase total enrollment to compensate

Sample Size Calculations

Before collecting data, we must answer:

  1. How precise do we want our estimates to be?
  2. How likely do we want to detect a meaningful difference?

These are two different goals:

  • precision [estimation]
  • power [hypothesis testing]

Sample Size and Power Calculation: Example

Background:

  • Tumor Mutational Burden (TMB) is the number of somatic mutations per megabase (mut/Mb)
  • In colorectal cancer, tumors with deficient mismatch repair (dMMR) often show higher TMB than proficient mismatch repair (pMMR) tumors

Research Task: - A colorectal cancer genomics team is deciding whether to invest in deeper sequencing for mismatch-repair testing - The group wants to know whether dMMR tumors have meaningfully higher Tumor Mutational Burden (TMB) than pMMR tumors, using TMB as a quantitative signal of DNA repair deficiency

Why this is a good power calculation example:

  • outcome is continuous (mut/Mb), typical of genomics summary measures
  • the effect size can be stated in biologically meaningful units (mut/Mb)

Power/Sample Size Questions:

Precision question [determine sample size needed to estimate parameter]:

  • How many dMMR tumor samples do we need so that our estimate of the mean TMB has a 95% confidence interval no wider than a prespecified margin [i.e.: ±2 mut/Mb]?

Power question [determine power to detect signal with sample size]:

How many tumors per group do we need so that we have 80% power to detect it at α = 0.05?

Key Definitions

parameter: fixed but unknown population value [i.e.: \(\mu_{dMMR}\)]

statistic: value calculated from the sample [i.e.: \(\bar{x}_{dMMR}\)]

standard error [SE]: variability of a statistic across repeated samples

α: false positive rate

  • Type I error - rejecting the null hypothesis when it is actually true
  • significance level is set to a fixed value: α=0.05

power = 1 − β:

  • probability that your study will correctly detect a real effect
  • β = probability of a Type II error [missing a real effect]

For a mean: \[ SE = \frac{\sigma}{\sqrt{n}} \]

  • \(\sigma\) = population standard deviation
  • \(n\) = sample size
  • \(n\) increases \(\rightarrow\) SE decreases

Example: Estimating Mean TMB

The following determines the sample size needed to estimate a mean given a desired margin of error

margin of error for a confidence interval is:

\[ ME = z \cdot \frac{\sigma}{\sqrt{n}} \]

Determine the following:

Define the variability

  • \(\sigma = 10\) mut/Mb
    • σ: estimated standard deviation of TMB from prior studies (mut/Mb)

Define margin of error

  • Desired margin of error = 2 mut/Mb

Confidence Level [z-score]:

  • 95% confidence: \(z = 1.96\)

Sample size formula:

  • calculate the minimum number of tumors required

\[ \left(\frac{z \sigma}{ME}\right)^2 \]

Example: Estimating Mean TMB

sigma <- 10
ME <- 2
z <- 1.96

n_precision <- (z * sigma / ME)^2

#round up to ensure adequate precision
ceiling(n_precision)
[1] 97

Interpretation:

  • study needs at least 97 tumors to estimate the mean TMB with a 95% confidence interval whose half-width is ±2 mut/Mb

  • if the study needs to be repeated study many times the estimated mean TMB would typically fall within 2 mut/Mb of the true population mean

Note:

Increasing sample size reduces uncertainty because

\[ SE = \frac{\sigma}{\sqrt{n}} \]

As \(n\) increases:

  • standard error decreases
  • confidence intervals become narrower
  • estimates become more precise

Summary

  • sampling distributions describe how a statistic behaves across repeated samples and provide the foundation for statistical inference.

  • sampling design determines validity

    • who the sample represents

    • whether estimates generalize to the population.

  • sample size determines uncertainty

    • larger samples → smaller standard errors

    • narrower confidence intervals

    • greater statistical power

The “inputs” to sample size planning

To calculate sample size, you must specify assumptions. These are not “math details”—they are the scientific commitments that determine feasibility.

For a two-group comparison, the typical inputs are:

  • Δ: smallest meaningful difference in means (mut/Mb)
  • σ: outcome standard deviation (mut/Mb)
  • α: false positive rate (Type I error)
  • power = 1 − β: probability of detecting Δ if it is truly present
  • design: independence vs clustering, missingness, unequal group sizes

You should interpret any final n as “n under these assumptions.”

Step 0: Declare assumptions for our example

We will assume prior data suggest:

  • Standard deviation of TMB: σ ≈ 10 mut/Mb
  • A biologically meaningful mean difference: Δ = 4 mut/Mb
  • Two-sided test, α = 0.05
  • Desired power = 0.80
  • Equal group sizes (dMMR and pMMR)

We will compute: - n for precision - n for power - achieved power under constraints - how n changes when Δ changes

Task A: Precision-based sample size (one mean)

Goal

Estimate μ_dMMR (mean TMB among dMMR tumors) with a 95% confidence interval whose half-width is at most ME.

The classical CI form [ {Y} z_{0.975}SE({Y}) ]

where:

  • ({Y}) is the sample mean
  • (z_{0.975}) for 95% confidence (large-sample normal critical value)
  • (SE({Y}) = /)

So the margin of error is:

[ ME = 1.96 ]

Solving for n:

[ n = ()^2 ]

Precision inputs for this study

We choose: - σ = 10 mut/Mb (from prior data) - target ME = 2 mut/Mb (we want estimates within ±2 mut/Mb)

Interpretation of ME in scientific terms: - If ME = 2, then the CI width is about 4 mut/Mb. - That is, we expect our estimate to be “within a few mut/Mb” of the population mean under repeated sampling.

z <- 1.96

How to read the precision output

The output is an integer n (after ceiling()), meaning:

  • minimum analyzable sample size needed to achieve the target ME

What the calculation is doing statistically:

  • It treats σ as known (or well-approximated).
  • It uses the normal critical value 1.96, which is appropriate when n is moderate/large.
  • The key scaling is (1/): to cut ME in half, you need ~4× the sample size.

Peck & Olsen emphasize this as a sampling distribution result: - variability of the sample mean shrinks with ().

Task B: Power-based sample size (two means)

Detect a difference in mean TMB between dMMR and pMMR tumors.

We define the scientific target as: [ = {dMMR} - {pMMR} = 4 ]

A standard two-sample comparison centers on the standardized effect size:

[ d = ]

Here: - Δ is in mut/Mb (biological units) - σ is in mut/Mb (variability) - d is unitless (signal-to-noise scale)

This “signal over noise” idea is emphasized in Practical Statistics for Data Scientists.

What assumptions the two-sample t-test uses

The classic two-sample t-test (as implemented in many power calculators) assumes:

  • independent tumors within each group
  • approximately normal sampling distribution of the mean difference
  • common within-group standard deviation (σ) or a pooled estimate
  • specified α and target power

Even when TMB is skewed, with moderate n the mean difference often has an approximately normal sampling distribution (CLT logic).

ISLR’s modeling perspective connects here: - inference relies on approximate normality of estimators (many procedures do).

R package used for power calculations

We use the pwr package.

What it provides: - standard power calculations for common tests (t-test, correlation, ANOVA, etc.) - consistent output objects that report n, power, effect size, and α

How to interpret it: - It returns the sample size required under the specified design and assumptions - It does not validate whether your σ or Δ assumptions are correct—that is your scientific responsibility.

R: power-based sample size (two-sample t-test)

library(pwr)

Delta <- 4
sigma <- 10
effect_size <- Delta / sigma  # Cohen's d

pwr_result <- pwr.t.test(d = effect_size,
                         power = 0.80,
                         sig.level = 0.05,
                         type = "two.sample",
                         alternative = "two.sided")

pwr_result

              d = 0.4
      sig.level = 0.05

How to read pwr.t.test() output

pwr.t.test() prints a structured summary. The key fields are:

  • d: standardized effect size (Δ/σ). Here 0.4 means “difference is 0.4 SDs.”
  • sig.level: α (false positive rate), usually 0.05
  • power: target probability of detecting Δ, here 0.80
  • n: required per group sample size for a two-sample design
  • alternative: one-sided vs two-sided (two-sided requires larger n)

Interpretation in this lecture: - If output reports n ≈ 100, that means ~100 tumors per group. - Total analyzable sample size would be ~200 tumors.

Why the “per group” detail matters

Many students misread power output as total n.

For type="two.sample" in pwr.t.test():

  • n is per group
  • total n ≈ 2n when groups are equal

This is a design statement: - power depends on the precision of the difference in means, which uses information from both groups.

What changes if the meaningful difference is smaller?

Now suppose the biology suggests that even Δ = 2 mut/Mb would be important.

Everything else stays the same (σ = 10, α = 0.05, power = 0.80).

New effect size: [ d = = 0.2 ]

This is half the signal, with the same noise.

Interpreting the “smaller Δ” output

You should expect n to increase substantially.

Reason (conceptual, from the formulas):

  • The standard error shrinks as (1/).
  • If Δ is cut in half, you need roughly 4× n to restore the same signal-to-noise separation.

This is one of the most important practical messages in sample size planning: - small, meaningful effects are expensive to detect unless σ is small.

How to interpret achieved power

If the output power is, for example, ~0.55–0.65, interpret it as:

  • Under the assumed Δ and σ, your study detects the effect only ~55–65% of the time.
  • That implies a ~35–45% chance of a false negative even when the effect exists.

Peck & Olsen’s framing: - A non-significant result in an underpowered study does not strongly support “no effect.” - It often reflects insufficient information.

Plot: power curve as n increases

A power curve shows how power changes as a function of sample size. This is a helpful design communication tool: it makes tradeoffs visible.

  pwr.t.test(d = 4/10,
             n = n,
             sig.level = 0.05,

})

plot(ns, power_curve, type = "l",

     ylab = "Power to detect Δ = 4 mut/Mb")
abline(h = 0.80, lty = 2)

How to read the power curve plot

n the plot:

  • x-axis: n per group
  • y-axis: power (probability of detecting Δ if true)
  • the curve rises because increasing n reduces the SE of the mean difference
  • the dashed line at 0.80 is the conventional target

How this supports decision-making (Practical Statistics perspective):

  • If you are below 0.80 at feasible n, you can:
  • increase n
  • accept lower power (with explicit consequences)
  • increase Δ (redefine “meaningful”)
  • reduce σ (better measurement, better design)
  • change the design (paired/repeated measures, covariate adjustment)

Design adjustments: turning “required n” into “planned enrollment”

The pwr calculation gives analyzable n under ideal assumptions.

Real biomedical studies commonly need inflation for:

  1. Missingness / QC failure
  2. Clustering / non-independence
  3. Unequal group sizes
  4. Multiple primary endpoints / multiplicity (if applicable)

We focus on the first two here.

Missingness / QC failure adjustment

If a fraction r of samples are unusable (dropout, assay failure, sequencing QC), then:

\[
  n_{enroll} = \frac{n_{required}}{1-r}
  \]

Example: - required n = 100 per group - expected unusable rate r = 0.10 (10%)

Then: [ n_{enroll} = ]

Interpretation: - you enroll 112 to end with ~100 analyzable tumors.

Clustering adjustment (design effect)

If observations are grouped (e.g., multiple centers) and within-center outcomes are correlated, independence is violated.

A common approximation is the design effect:

\[
  DE = 1 + (m-1)\rho
  \]

where: - (m) = average cluster size (tumors per center) - () = intra-cluster correlation (ICC)

Effective sample size: [ n_{effective} ]

Interpretation: - correlation makes some observations partially redundant - you need more total tumors to achieve the same information as SRS

Quick calculator: dropout + design effect

  n_required <- 100      # per group from power calculation
  dropout <- 0.10
  m <- 20                # avg tumors per center
  ICC <- 0.02
  
  DE <- 1 + (m - 1) * ICC
  n_enroll <- ceiling((n_required * DE) / (1 - dropout))
  
  list(design_effect = DE,
       enroll_per_group = n_enroll,
       enroll_total = 2 * n_enroll)

How to interpret the adjustment output

This output reports:

- **design_effect**: inflation factor due to clustering

Key message: - design and missingness reduce effective information - planning is about achieving target information, not simply raw n

Across the three books, the shared logic is:

- sampling variability creates uncertainty

Stratified sampling: a design choice that can reduce variance

If variability differs across subgroups, stratification can:

- ensure each subgroup is represented
  • reduce standard errors for overall estimates

Example: two ancestry groups with different biomarker distributions.

Code example: SRS vs stratified sampling standard error

  #| label: l3-stratified-se6
  #| echo: true
  
  set.seed(29)
  
  # population with two strata and different means/variances
  pop2 <- tibble(
    strata = sample(c("A", "B"), size = N, replace = TRUE, prob = c(0.7, 0.3))
  ) |>
    mutate(biomarker = if_else(strata == "A",
                               rnorm(N, mean = 10, sd = 2),
                               rnorm(N, mean = 14, sd = 4)))
  
  true_mean2 <- mean(pop2$biomarker)
  
  srs_once <- function(n){
    mean(sample(pop2$biomarker, n, replace = FALSE))
  }
  
  strat_once <- function(n){
    # proportional allocation (same sampling fractions as population)
    nA <- round(n * mean(pop2$strata == "A"))
    nB <- n - nA
    mean(c(sample(pop2$biomarker[pop2$strata == "A"], nA, replace = FALSE),
           sample(pop2$biomarker[pop2$strata == "B"], nB, replace = FALSE)))
  }
  
  n <- 200
  B <- 2000
  
  srs_est  <- replicate(B, srs_once(n))
  str_est  <- replicate(B, strat_once(n))
  
  tibble(
    design = c("SRS", "Stratified (proportional)"),
    mean_est = c(mean(srs_est), mean(str_est)),
    sd_est   = c(sd(srs_est), sd(str_est))
  )
# A tibble: 2 × 3
  design                    mean_est sd_est
  <chr>                        <dbl>  <dbl>
1 SRS                           11.2  0.231
2 Stratified (proportional)     11.2  0.196

Interpreting the stratified result

If `sd_est` is smaller for the stratified design, you gained precision **without** increasing $$n$$.

That tradeoff matters in practice because:

- sequencing more samples costs money
  • recruiting participants costs time
  • but redesigning sampling can sometimes deliver precision “for free”

A practical sample size lens: margin of error

For many estimators, uncertainty is roughly:

\[ \text{estimate} \pm z_{0.975}\times SE(\text{estimate}) \]

For a proportion:

\[ SE(\hat{p}) \approx \sqrt{\frac{\hat{p}(1-\hat{p})}{n}} \]

So the margin of error shrinks like $$1/\sqrt{n}$$ (slowly).

Code example: margin of error for a proportion across \[n\]

  #| label: l3-moe-prop5
  #| fig-width: 6
  #| fig-height: 4
  #| echo: false
  
  p <- 0.15
  ns <- seq(50, 3000, by = 50)
  
  moe <- tibble(
    n = ns,
    se = sqrt(p*(1-p)/n),
    moe_95 = 1.96 * se
  )
  
  ggplot(moe, aes(x = n, y = moe_95)) +
    geom_line() +
    labs(
      title = "Margin of error shrinks as 1/sqrt(n)",
      subtitle = "Example: 95% CI half-width for a proportion with p = 0.15",
      x = "Sample size (n)",
      y = "95% margin of error"
    ) +
    theme_bw()

Power: sample size as “detectability”

Power answers:

> If there is a real effect of a given size, how often will our test detect it?

Power depends on:

- effect size (clinically meaningful difference)
  • variability / event rate
  • alpha level (false positive tolerance)
  • sample size

Takeaways you can use immediately

- **Sampling design defines the repeated-sampling world** your inference lives in.
  • Bias is a design/data problem; it can persist even when \[n\] is large.
  • Variance is a sample size problem; it shrinks like \[1/\sqrt{n}\].
  • Sample size planning is really planning for:
    • desired precision (margin of error) and/or
  • desired power (detectability of an effect)
